Security is not an afterthought at Tamperline.us — it’s foundational. Your brewery’s inventory data, customer records, and business information are protected by industry-standard security practices. Here’s exactly how we keep your data safe.

01

Our Commitment

We are committed to maintaining the confidentiality, integrity, and availability of your business data. This page describes our current security practices. We continuously improve our security posture and will update this page when significant changes are made.

✅ Tamperline.us has maintained 99.9% uptime with zero confirmed data breaches since launch.
02

Data Encryption

  • All data encrypted in transit using TLS 1.2+
  • All data encrypted at rest using AES-256
  • Database backups encrypted using the same standards
  • Passwords hashed using bcrypt with per-user salts
  • API tokens are hashed and never stored in plain text
03

Access Control

  • Role-based access control (RBAC) for all user accounts
  • Multi-factor authentication (MFA) available for all accounts
  • Session tokens expire after 30 days of inactivity
  • Admin access to production systems is audited and logged
  • Principle of least privilege applied to all internal systems
04

Infrastructure & Hosting

Tamperline.us is hosted on industry-leading cloud infrastructure with multiple redundancy layers.

  • Hosted on AWS (US-East region)
  • Automated daily backups with 30-day retention
  • 99.9% uptime SLA with active monitoring 24/7
  • DDoS protection via Cloudflare
  • Isolated production/staging environments
  • Automated vulnerability scanning on dependencies
05

Compliance

  • GDPR-compliant data handling for EU users
  • CCPA-compliant for California residents
  • SOC 2 Type II audit in progress
  • Data Processing Agreements (DPA) available on request
  • We do not sell your data to third parties — ever
06

Vulnerability Reporting

We take vulnerability reports seriously and appreciate responsible disclosure.

⚠️ Do not publicly disclose potential vulnerabilities before we’ve had a chance to investigate and fix them.
  • Email: ravi@tamperline.us
  • Include: Description of the issue, steps to reproduce, potential impact
  • Response time: We will acknowledge within 48 hours
  • We do not currently operate a bug bounty program but recognize reporters publicly (with permission)
